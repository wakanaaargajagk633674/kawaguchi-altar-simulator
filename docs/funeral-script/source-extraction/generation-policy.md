# AI生成 / 固定テンプレート 切り分け方針

> 進行案内をAIに自由生成させるとミス（式次第の取り違え・誤った作法案内）が起きやすい。
> **進行・案内＝固定テンプレート**、**物語・人物描写＝AI生成** を原則とする。
> 抽出資料の実態（台本は定型・ナレは文例＋取材差し替え）とも一致。

## 原則
1. **固定テンプレート（fixed）**：式の安全性に関わる進行・案内。変数差し込みのみ。AIに文を作らせない。
2. **AI生成（ai）**：故人の物語・人柄・情緒表現。取材値（入力フォームA群）を素材に生成。
3. **ハイブリッド（hybrid）**：固定の骨組みにAI生成の一文を挿し込む（例：開式前ナレの導入は固定枠＋故人紹介はAI）。

## AI生成に向いている部分　[ai]

| 部分 | 素材（入力A群）| 根拠（出典 No.）|
|------|------|------|
| 故人略歴ナレーション | 生年月日/出身地/学歴/職歴/時代背景 | No.039, 045, 011 |
| 人柄紹介 | 人柄/エピソード/口癖 | No.031, 033, 034 |
| 遺影写真紹介 | 遺影の由来（いつ/どこで/なぜ）| No.002, 029, 037 |
| 趣味・エピソード紹介 | 趣味/好きな物/活動 | No.004, 023, 028 |
| 家族への想い | 家族構成/エピソード | No.003, 006, 016 |
| 地域活動・功労紹介 | 地域活動/功労/職務 | No.012, 017, 025 |
| 無宗教式のメインナレーション | 略歴＋人柄＋物語 | No.044–045 |
| 閉式前ナレーション | 全体の情緒的まとめ | No.039, 015 |

→ AI生成には以下を**プロンプト制約**として渡す（資料由来の作法）：
- 故人の言葉は『　』で引用し前後に間（改行/句点）を置く。No.001, 005
- 名詞列挙は流さず一つずつ（読点で区切る短文連結）。No.005, 007
- 「死亡/急死」等の直接表現を避け「ご逝去/永眠/ご生涯を閉じられました」。No.020, 040
- 文末「〜だそうです」は司会者＝第三者の立場として許容。No.023
- 取材で得た事実のみを使い、創作・誇張をしない（不明箇所は生成しない）。

## ハイブリッド　[hybrid]
- 開式前ナレーション：固定の導入/結び枠 ＋ AI生成の故人紹介。No.039–040
- 出棺案内：固定の進行文 ＋ AI生成の一文（趣味に因む見送り）。No.052(㊾), 026

---

## 安全設計メモ
- AI生成テキストは**進行・作法に関する文言を含めない**ようガードする（焼香順・宗派作法・初七日の有無などは fixed のみが扱う）。
- 生成結果は司会者が**必ず校正**する前提（資料の心得：声に出して読む。No.043）。アプリ上で編集可能に。
- 宗派・式種別・各有無フラグは fixed テンプレートの選択に使い、AI には文体・素材のみ委ねる。
- 忌み言葉チェックは別途ルール（要追補。資料に明示リストなし）として AI 出力後に適用すると安全。
