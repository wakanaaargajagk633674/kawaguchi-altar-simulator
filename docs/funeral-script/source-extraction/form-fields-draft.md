# 入力項目ドラフト（台本作成アプリ）

> 予定入力項目を、抽出資料（取材シート No.054 / 司会シート No.047 / 喪家確認シート No.056 / 取材質問 No.053）と突き合わせて整理。
> 「採用＝予定どおり / 追加＝資料から新規提案 / 任意」を明記。変数名は `app-template-draft.md` と対応。

## 不要・統合の提案
- 予定項目の「導師、寺院、司式者情報」は粒度が粗いので `templeName`/`buddhistSect`/`officiantName`/`menholderTemple(お手次寺)` に分解推奨（No.056）。
- 「焼香、献花、献灯の有無」は1項目でなく**3つの独立フラグ**に（無宗教式で選択式 No.046）。

## 設計メモ
- 入力は **C（式の事実）** と **A（取材＝物語素材）** を分離。C は固定テンプレ差し込み、A は AI 生成の素材（`generation-policy.md` 参照）。
- 個人情報を含むため、保存時は変数（テンプレート）と値を分離して扱う設計が望ましい（本番画面に抽出全文・画像本文を出さない方針）。
